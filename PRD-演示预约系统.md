```
│                                        │

│ ☑ 张三  ☐ 李四  ☐ 王五              │
│                                        │
│       [取消]    [确认预约]              │
└────────────────────────────────────────┘
```

**预约检查逻辑**：
- 遍历每一选择的设备，遍历时间段内每一天
- **设备**：当日该设备可用单元数 ≥ 用户选的数量？
- **人员**：当日空闲人数 ≥ 用户勾选人数？
  - 人员占用仅依据管理员安排的任务（A），不依据用户勾选（B）
- 任一天任一条件不满足 → 弹窗提示：`5月19日 频谱分析仪不足（仅剩2台），请调整`

**提交后**：
- 多个设备各自分配可用单元 → status 改为 occupied
- 一条预约记录写入 equipment_reservations（含设备选择、unit_ids、勾选人员）
- 人员状态不因用户勾选而改变

### 4) 我的预约
- 表格：设备 | 数量 | 日期范围 | 国家 | 用途 | 状态
- 操作：仅 **取消**（无归还功能）
- 取消后：对应设备单元释放 → status 改为 available

## 四、预约约束详细逻辑

```
用户提交预约请求：
  选中的设备类型: 频谱分析仪
  数量: 2
  人员: 张三、李四（勾选）
  时间段: 5/18 - 5/20

遍历每一天 (5/18, 5/19, 5/20):

  【设备检查】
  当日可用设备单元 = 该设备总单元 - 当日被其他active预约占用的单元
  if 可用单元 < 用户需求数量(2):
    失败 → 提示 "5/19 频谱分析仪不足（仅剩1台）"

  【人员检查】
  当日空闲人数 = 总人数 - 当日有任务安排的人数
  if 空闲人数 < 用户勾选人数(2):
    失败 → 提示 "5/19 人员不足（仅1人空闲）"

全部通过 → 预约成功
```
